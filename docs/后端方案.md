## 🎯 决策框架：先问自己三个问题

| 问题 | 你的答案 | 推荐方向 |
|-----|---------|---------|
| **数据要存多久？** | 临时/可丢 → 长期重要 | 内存/Local → 正式数据库 |
| **需要复杂查询吗？** | 简单 CRUD → 关联查询/聚合 | KV 存储 → SQL/NoSQL |
| **愿意管服务器吗？** | 不想管 → 可以接受 | Serverless → 自托管 |

## 方案一：无后端，Next.js 直接连数据库（推荐起步）

Next.js 的 Server Actions / API Routes 可以直接操作数据库，无需单独后端服务。

### 1. 轻量级：Vercel/Netlify 原生方案

| 服务 | 类型 | 免费额度 | 适用场景 |
|-----|------|---------|---------|
| **Vercel KV** | Redis | 256MB | 缓存、Session、计数器 |
| **Vercel Postgres** | PostgreSQL | 60h 计算/月 | 关系型数据、用户系统 |
| **Vercel Blob** | 对象存储 | 250MB | 图片、文件上传 |
| **Neon** | Serverless Postgres | 500MB 存储 | 长期项目，可迁移 |
| **Supabase** | Postgres + 实时 | 500MB | 需要实时订阅、Auth |

```typescript
// app/actions.ts - Server Action 直接操作数据库
'use server'

import { sql } from '@vercel/postgres'

export async function createUser(formData: FormData) {
  const name = formData.get('name') as string
  const email = formData.get('email') as string
  
  // 直接 SQL，无需 API 层
  const { rows } = await sql`
    INSERT INTO users (name, email) 
    VALUES (${name}, ${email}) 
    RETURNING id
  `
  return rows[0]
}
```

```typescript
// 客户端调用，像调用本地函数
'use client'

import { createUser } from './actions'

export default function Form() {
  return <form action={createUser}>...</form>
}
```

import { MongoClient } from 'mongodb'

const uri = process.env.MONGODB_URI!

## 方案二：需要独立后端（复杂业务时）

当 Next.js 需要调用外部 API，或业务逻辑太复杂需要分离：

### 架构模式

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   前端      │ ───→ │  Next.js    │ ───→ │  独立后端   │
│ (App Router)│      │  (BFF层)    │      │  (复杂逻辑) │
└─────────────┘      └─────────────┘      └─────────────┘
                            ↓
                     ┌─────────────┐
                     │  数据库     │
                     │ (同方案一)  │
                     └─────────────┘
```

### 后端技术选型

| 场景 | 推荐方案 | 部署 |
|-----|---------|------|
| 快速全栈 | **Next.js API Routes** 自己写 | Vercel |
| 需要队列/定时任务 | **Inngest** + Next.js | Vercel 集成 |
| 重业务逻辑 | **Hono/Fastify** (Node) | Railway/Render |
| Python 生态 | **FastAPI** | Railway/Render |
| 边缘函数 | **Cloudflare Workers** | Cloudflare |
| 无服务器函数 | **AWS Lambda** | Serverless Framework |

```typescript
// app/api/webhook/route.ts - Next.js 作为 BFF 转发到后端
import { NextRequest, NextResponse } from 'next/server'

export async function POST(req: NextRequest) {
  const body = await req.json()
  
  // 验证、转换后转发到独立后端
  const response = await fetch(`${process.env.BACKEND_URL}/process`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${process.env.BACKEND_KEY}` },
    body: JSON.stringify({
      userId: body.userId,
      // 数据清洗...
    }),
  })
  
  return NextResponse.json(await response.json())
}
```

## 方案三：特定场景方案

### 纯静态托管（GitHub Pages）+ 外部数据

GitHub Pages 不支持 Serverless Functions，需要：

| 数据需求 | 解决方案 |
|---------|---------|
| 只读数据 | 构建时抓取，生成静态页面 (`getStaticProps`) |
| 少量动态 | **GitHub Issues** 作为 CMS (如 Giscus 评论) |
| 用户数据 | **Supabase** / **Firebase** 客户端直连 |
| 表单提交 | **Formspree** / **Getform** / **Airtable** |

// 构建时抓取，纯静态
// app/blog/page.tsx
export async function generateStaticParams() {
  const posts = await fetch('https://api.example.com/posts')
    .then(r => r.json())
  
  return posts.map((post: any) => ({
    slug: post.slug,
  }))
}

// 或完全静态 JSON
import posts from '@/data/posts.json' // 本地 JSON 作为数据库

      .channel('orders')
      .on('postgres_changes', 
        { event: 'INSERT', schema: 'public', table: 'orders' },
        (payload) => {
          console.log('New order:', payload.new)
          // 更新 UI...

      )
      .subscribe()
    
    return () => { supabase.removeChannel(channel) }
  }, [])
}

## 关键代码模式：Server Actions 最佳实践

```typescript
// lib/actions.ts - 所有数据库操作集中管理
'use server'

import { revalidatePath } from 'next/cache'
import { db } from './db'
import { posts } from './schema'

export async function createPost(formData: FormData) {
  try {
    const title = formData.get('title') as string
    const content = formData.get('content') as string
    
    await db.insert(posts).values({ title, content })
    
    revalidatePath('/posts') // 清除缓存
    return { success: true }
  } catch (error) {
    return { success: false, error: '创建失败' }
  }
}

export async function deletePost(id: number) {
  await db.delete(posts).where(eq(posts.id, id))
  revalidatePath('/posts')
}
```
